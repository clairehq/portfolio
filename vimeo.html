Vimeo App Ethical Evaluation

  • Date: January—April, 2019

  • My Role: user research, problem analyst, user testing

  • Group Members: Kim Hu, Claire Li, Kunlu Li

  • Tools: Figma, Sketch

Context

This project is about evaluating and analyzing any problems related to user interface and user experience found in Vimeo iOS app, and then design a revision solution for each problems found. The purpose of this project is about practicing principles of interactive design and user testing ability.

Persona

we created a typical user persona.

Usability Test

First, We came out with some performing tasks that users normally perform when using Vimeo.

1. Attempt to watch a video with captions on.

2. Attempt to like a video.

3. Attempt to hide a video that not interested.

4. Attempt to turn off notifications from a channel.

5. Attempt to find the oldest video in a channel.

6. Attempt to change profile photo.

7. Attempt to delete account.

Why choose those tasks?

Those tasks are typical task flows as it is on the tasks that users need to perform to achieve their goals. Also, it covers a wide range of users’ activities that could happen in this App.

Principles broken

Based on our tasks, we found several problems within the app that broke the principles set by Humane by Design and Artifact.

  • Extended Solve: A design solution that focuses on an excluded disabled or impaired group of users can benefit all users.
  • In Control: Give people the control they need to manage the algorithms that shape their experiences.
  • Allow for Personalization: Allow users to customize from whom, when and how they can receive notifications to minimize unwanted interruptions.

We then rated them based on Jacob Nielson's Severity Ratings for Usability Problems.

Severity Rating

🔵 If users do not want to receive notifications from a specific person that they follow, they cannot just turn off notifications from that person.

🔵 If a viewer has a severe hearing impairment or in a loud environment and cannot hear the sounds clearly even with the maximum volume, he/she may need to turn on the caption. However, this function is not provided in the current version of Vimeo.

🔵 If viewers want to watch the newest video, they have to compare the uploaded time by themselves because the application does not offer a filter that allows the viewers to sort videos by time.

🔵 If a viewer does not like a video, he/she cannot hide it because this function is not offered.

🔵 If viewers like a video and wish to add in favorite, they cannot simply press the heart shape. Instead, they have to open the menu to find the function.

Design Revision

We made three revisions for the problems we found.

1. Hide uninterested Video

I added a function of hiding uninterested videos. Since not all of the videos on the list are suitable for every user, we think it may be beneficial to incorporate a feature that allows users to hide videos that they don’t want to watch. This feature offers users the control they need for deciding what videos they want to appear, or not to show, in further searches. Users can use this function in many ways, such as in the case when specific videos are not suitable for a user to watch, or when the users have seen the video already. Besides, users can also recover from their actions if they hide videos by accident.

2. Change Notification Frequency

I added a function that allows users to change how frequently they want to receive notifications from creators they follow. They can turn on or off of any creator’s notification completely, or they can set the frequency to "sometimes." As the pictures showed, users can click the notification icon and change the notification frequency. There are three options available: receive all notifications, receive some notifications, and receive no notifications.

3. Sort Video by Uploaded Date

The last revision I made is adding a search filter for sorting videos based on uploaded time and popularity. This feature can shorten users’ video-searching time. For example, if users want to check the oldest video, they don’t need to scroll down to the very end. Instead, they can tap on the filter and choose the “Uploaded Date (Oldest)” option. After selecting this option, the app will sort the video according to this preference.
